package com.example.ecommerceplatform.model;

import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import jakarta.persistence.Table;

@Entity
@Table(name = "users") // Optional: Specifies the table name
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY) // Auto-incremented primary key
    private Long id;

    private String username;
    private String email;
    private String password;
    private String role; // E.g., Admin, Buyer, Seller

    // Constructors
    public User() {
    }

    public User(String username, String email, String password, String role) {
        this.username = username;
        this.email = email;
        this.password = password;
        this.role = role;
    }

    // Getters and Setters
    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    }

    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getEmail() {
        return email;
    }

    public void setEmail(String email) {
        this.email = email;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }

    public String getRole() {
        return role;
    }

    public void setRole(String role) {
        this.role = role;
    }

    // toString Method (Optional, useful for debugging)
    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", username='" + username + '\'' +
                ", email='" + email + '\'' +
                ", role='" + role + '\'' +
                '}';
    }
}

Explanation of Annotations

  1. @Entity:

    • Marks this class as a JPA entity to be mapped to a database table.

  2. @Table(name = "users"):

    • Specifies the name of the table in the database. If omitted, the table name defaults to the class name (User).

  3. @Id:

    • Marks the field as the primary key.

  4. @GeneratedValue(strategy = GenerationType.IDENTITY):

    • Specifies the primary key generation strategy:
      • GenerationType.IDENTITY: Auto-incremented by the database.

  5. @Column (Optional):

    • You can annotate each field with @Column to customize column names, constraints, and settings (e.g., @Column(name = "user_email", nullable = false)).

This class generates a table with the following structure:

The @Id annotation in a JPA entity is provided by the Jakarta Persistence API (formerly Java Persistence API). Specifically, it comes from the jakarta.persistence package starting with Jakarta EE 9.

If you are using Spring Boot 3.x or later, you should use the Jakarta package for all JPA annotations because Spring Boot 3 transitioned to Jakarta EE.

Correct Import for @Id

For modern Spring Boot versions (3.x and later):

java
import jakarta.persistence.Id;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;

For Older Spring Boot Versions (2.x and earlier)

If you are using Spring Boot 2.x, the JPA annotations will come from the javax.persistence package:

java
import javax.persistence.Id;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;

How to Check Which Package to Use

  1. Spring Boot Version:

    • Spring Boot 3.x → Use jakarta.persistence.
    • Spring Boot 2.x → Use javax.persistence.

  2. Maven or Gradle Dependency:

    • Spring Boot 3.x uses jakarta.persistence through the jakarta.persistence-api dependency.
    • Spring Boot 2.x uses javax.persistence through the javax.persistence-api dependency.

To establish relationships between entities in your e-commerce project, you will use JPA annotations like @ManyToOne, @OneToMany, and @ManyToMany. These relationships help define how the entities are connected and how the database tables should reference each other through foreign keys.

Here’s how and where to add relationships in your entities:

1. Relationships Between Entities

User and Order (One-to-Many)

  • A user can have multiple orders.
  • Add the relationship in both the User and Order classes.

Order and Product (Many-to-Many)

  • An order can include multiple products, and a product can be part of multiple orders.
  • This will require a join table (e.g., order_products).

User and Cart (One-to-Many)

  • A user can have multiple items in their cart.
  • Each cart item belongs to one user.

Cart and Product (Many-to-One)

  • A cart item refers to a single product.

Explanation of Annotations

  1. @OneToMany and @ManyToOne:

    • @OneToMany is used on the parent entity (e.g., User for Order).
    • @ManyToOne is used on the child entity (e.g., Order for User).

  2. @ManyToMany:

    • Defines a many-to-many relationship between two entities (e.g., Order and Product).
    • Requires a join table, defined using @JoinTable.

  3. @JoinColumn:

    • Specifies the foreign key column in the table.

  4. cascade = CascadeType.ALL:

    • Ensures that related entities are persisted or deleted when the parent entity is.

  5. orphanRemoval = true:

    • Automatically deletes child entities if they are removed from the relationship.
